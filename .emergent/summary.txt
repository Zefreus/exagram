The user wants to build a single-purpose multi-tenant SaaS web application called Exagram for the Brazilian market. The application will analyze blood count exams (hemograma).

**PRODUCT REQUIREMENTS:**
-   **Target User**: Individuals who have received blood exam results and want to understand them before a doctor's appointment.
-   **Architecture**: Multi-tenant with 'Admin' and 'Tenant' roles. Admins manage tenants, and each tenant has a single user account.
-   **Tech Stack**:
    -   Frontend: Next.js with TypeScript and Tailwind CSS.
    -   Backend: Next.js API routes.
    -   Database: External MySQL.
    -   AI: Anthropic Claude API for exam analysis and text extraction from PDFs/images.
    -   Payments: Stripe for one-time credit purchases and an optional subscription.

-   **Billing Model**: A credit-based system with one-time purchase packages and an optional monthly subscription. New tenants get one free credit.
-   **Admin Dashboard**: For managing tenants, specialists, and viewing aggregated system metrics. Stripe coupons should be managed via API.
-   **Tenant Dashboard**: For viewing exam history, credit balance, and purchasing more credits.
-   **LGPD Compliance**: Strict adherence to Brazilian data protection law, including explicit consent for sensitive health data, user rights (access, export, delete), and data minimization.
-   **UI/UX**: Clean, warm, and human-centric design in Brazilian Portuguese. Specific colors and fonts are provided.
